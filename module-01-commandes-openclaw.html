1
Avant de commencer
Comprendre ce qu'on fait dans cette étape en termes simples

On va louer un ordinateur distant (appelé VPS) et y préparer le terrain pour installer OpenClaw.

🔒
Pourquoi un ordinateur distant ? Parce qu'OpenClaw a accès à TOUT sur la machine où il tourne. Si vous l'installez sur votre Mac ou PC perso, il pourrait accéder à vos photos, vos fichiers bancaires, vos mots de passe... On ne veut pas ça.

🔤 Vocabulaire à connaître

VPS — Un ordinateur qu'on loue à distance, comme un Airbnb pour machines. Il tourne 24h/24 dans un datacenter.
Ubuntu — Le système d'exploitation qu'on installe sur le VPS. Comme Windows ou macOS, mais gratuit et fait pour les serveurs.
Terminal — La fenêtre noire où on tape des commandes. Comme si vous envoyiez des SMS à l'ordinateur : vous tapez une instruction, il l'exécute.
SSH — La connexion sécurisée entre votre ordinateur et le VPS. Un tunnel privé entre les deux machines.
🧘
Dans cette étape, chaque sous-étape est isolée. Prenez votre temps, il n'y a pas de chrono.
2
Créer un VPS Hostinger
L'ordinateur distant qui fera tourner OpenClaw 24h/24
💡 Pourquoi ?

C'est l'ordinateur distant qui fera tourner OpenClaw 24h/24 à votre place, même quand votre Mac/PC est éteint.

Rendez-vous sur Hostinger et créez un VPS KVM2 minimum (2 Go de RAM, environ 9,59 €/mois).

Une fois votre VPS créé, allez dans la section VPS de votre tableau de bord Hostinger.

✅ Si vous voyez votre VPS dans le tableau de bord avec un statut "Running", c'est bon — passez à l'étape suivante !
💡
Le KVM2 suffit largement pour OpenClaw avec un cerveau IA en ligne (Claude, GPT, etc.). Si vous voulez faire tourner un modèle IA directement sur le serveur (Ollama), prenez un KVM4 ou plus — mais ce n'est pas nécessaire pour débuter.
⚠️
Vous avez déjà un VPS Hostinger avec N8N ? Prenez-en un deuxième séparé. Les deux outils peuvent interférer entre eux sur la même machine.
3
Installer Ubuntu sur le VPS
Le système d'exploitation de votre serveur
💡 Pourquoi ?

Ubuntu est le système d'exploitation (comme Windows ou macOS) qu'on installe sur le VPS. C'est gratuit, fiable, et c'est ce qu'utilise OpenClaw.

Dans votre VPS Hostinger :

  1. Allez dans Paramètres du serveur
  2. Cliquez sur Système d'exploitation et panel
  3. Sélectionnez Ubuntu 24.04 LTS (c'est le choix recommandé)
  4. Ne choisissez PAS Docker, Coolify, ou un panel — juste Ubuntu nu

L'installation prend environ 5-7 minutes. Attendez que le statut repasse à "Running".

✅ Si vous voyez "Ubuntu 24.04" dans les infos de votre VPS et que le statut est "Running", c'est bon !
🔴
Attention : Cette opération efface TOUT sur le VPS. Si vous aviez déjà N8N, Docker, ou autre chose installé dessus, tout sera supprimé. Faites une sauvegarde si nécessaire.
4
Créer et noter le mot de passe root
La clé d'entrée de votre VPS
💡 Pourquoi ?

Ce mot de passe est la clé d'entrée de votre VPS. Vous en aurez besoin plusieurs fois dans les étapes suivantes.

Hostinger va vous demander de créer un mot de passe. Cliquez sur Générer puis copiez-le immédiatement dans un endroit sûr (l'app Notes de votre téléphone, un document, 1Password, etc.).

Cliquez sur Confirmer pour lancer l'installation.

🔑
Le mot de passe invisible : Dans les prochaines étapes, quand vous taperez ce mot de passe dans un terminal, rien ne s'affichera à l'écran. Pas d'étoiles, pas de points, rien du tout. C'est normal ! C'est une mesure de sécurité de Linux. Tapez votre mot de passe "à l'aveugle" et appuyez sur Entrée.
🔴
Notez ce mot de passe MAINTENANT ! Prenez une photo, collez-le dans vos notes, envoyez-le-vous par email... Vous en aurez besoin au moins 3 fois dans les prochaines minutes.
5
Ouvrir le Terminal Hostinger
La fenêtre pour envoyer des commandes au serveur
💡 Pourquoi ?

Le terminal, c'est la fenêtre où on va envoyer des commandes au serveur. Pensez-y comme une messagerie instantanée avec l'ordinateur : vous tapez une instruction, il l'exécute.

Une fois qu'Ubuntu est installé (statut "Running"), cliquez sur le bouton Terminal dans l'interface Hostinger. C'est généralement un bouton en haut à droite ou dans le menu latéral.

Vous devriez voir apparaître une fenêtre noire avec du texte et un curseur clignotant. C'est votre terminal serveur — vous êtes "à l'intérieur" de votre VPS.

✅ Si vous voyez un écran noir avec du texte (par exemple root@srv...) et un curseur qui clignote, c'est parfait !
💡
Ce terminal est celui du serveur Hostinger (votre VPS). Plus tard, on ouvrira aussi le terminal de votre ordinateur (Mac ou Windows). Ne les confondez pas — on vous préviendra à chaque fois lequel utiliser.
6
Lancer le script d'installation
Un seul copier-coller qui automatise tout le travail technique
💡 Pourquoi ?

Ce script automatise tout le travail technique : il crée un utilisateur sécurisé, installe les outils nécessaires (navigateur, bureau distant, etc.). Sans ce script, il faudrait taper une dizaine de commandes une par une.

📋 Comment copier-coller dans le terminal
  • Cliquez sur le bouton Copier ci-dessous (il copie la commande automatiquement)
  • Allez dans le terminal Hostinger
  • Pour coller : faites un clic droit > Coller, ou utilisez Ctrl+Shift+V (pas Ctrl+V tout seul — c'est différent dans un terminal !)
  • Appuyez sur Entrée pour lancer
Terminal Hostinger (en root) 
curl -fsSL https://openclaw-plateform.shubham-sharma.com/setup.sh -o /tmp/setup.sh && bash /tmp/setup.sh
🔑
Le script va vous demander de créer un mot de passe pour l'utilisateur openclaw. Choisissez le même mot de passe que celui de l'étape 4 pour simplifier. Rappel : le mot de passe est invisible quand vous le tapez. On vous le demandera deux fois (la 2e fois pour confirmer).
💡
Quand il demande "Would you like to enter a view-only password?" (mot de passe en lecture seule), tapez n puis Entrée.
✅ Si vous voyez "Setup completed" ou que le script se termine sans erreur rouge, c'est gagné !
ℹ️
Le mot de passe VNC sera tronqué à 8 caractères — vous verrez un message "Password truncated to the length of 8". C'est normal, pas une erreur !
7
Se connecter en SSH depuis votre ordi
Créer un tunnel privé entre votre Mac/PC et le serveur
💡 Pourquoi ?

Jusqu'ici, on utilisait le terminal de Hostinger (dans le navigateur). Maintenant, on va connecter votre ordinateur directement au VPS avec une connexion sécurisée appelée SSH. C'est comme créer un tunnel privé entre votre Mac/PC et le serveur.

🔄
ATTENTION : On change de terminal !
Fermez le terminal Hostinger (dans le navigateur). Maintenant, ouvrez le terminal de votre ordinateur.
Cmd+Espace → tapez "Terminal" → ouvrez-le 
ssh -L 6080:localhost:6080 openclaw@VOTRE_IP
Menu Démarrer → tapez "PowerShell" → ouvrez-le 
ssh -L 6080:localhost:6080 openclaw@VOTRE_IP
Configuration PuTTY 
# 1. Ouvrir PuTTY
# 2. Session → Host Name : VOTRE_IP — Port : 22
# 3. Menu gauche : Connection → SSH → Tunnels
#    Source port : 6080
#    Destination : localhost:6080
#    → Cliquer "Add"
# 4. Revenir dans Session → cliquer "Open"
# 5. Login : openclaw + votre mot de passe

Où trouver votre IP ? Dans le tableau de bord Hostinger > votre VPS. C'est un numéro qui ressemble à 31.97.116.221.

💡
Quand il demande "Are you sure you want to continue connecting?" → tapez yes et appuyez sur Entrée. Ensuite, tapez le mot de passe de l'étape 4. Rappel : le mot de passe est invisible.
✅ Si vous voyez openclaw@srv-quelquechose au début de la ligne, bravo ! Vous êtes connecté. Le tunnel SSH est en place.
🎓
Astuce pour les prochaines fois : la touche flèche haut ↑ dans le terminal rappelle les commandes précédentes. Vous n'aurez pas à retaper toute la commande SSH — juste appuyer sur ↑ et Entrée !
🔍 Comment savoir où vous êtes ?

Regardez le début de la ligne dans le terminal :

  • openclaw@srv... = vous êtes sur le serveur (c'est ce qu'on veut)
  • votre-nom@MacBook... ou C:\Users\... = vous êtes sur votre ordi (il faut vous connecter en SSH)
8
Accéder au bureau distant (VNC)
Voir un vrai bureau sur votre serveur — optionnel
💡 Pourquoi ?

Le VNC vous permet de voir un vrai bureau (comme Windows/macOS) sur votre serveur, directement dans votre navigateur. C'est utile pour voir les fichiers de configuration d'OpenClaw, mais ce n'est pas obligatoire.

👉
Si le VNC ne marche pas, ce n'est PAS grave ! Vous pouvez faire 100% des étapes suivantes sans le VNC, uniquement via le terminal SSH et Telegram. Le VNC est un bonus confort.

Pour y accéder :

  1. Votre connexion SSH doit être active (terminal ouvert avec openclaw@srv...)
  2. Ouvrez Google Chrome sur votre ordinateur
  3. Tapez l'adresse ci-dessous dans la barre d'adresse
  4. Cliquez sur Connect et tapez le mot de passe de l'étape 4
Adresse web — collez dans votre navigateur, PAS dans le terminal 
http://localhost:6080/vnc.html
✅ Si vous voyez un bureau avec des icônes (comme un écran Windows/Mac mais en plus simple), c'est gagné ! Astuce : appuyez sur Ctrl+H dans le gestionnaire de fichiers pour voir les dossiers cachés d'OpenClaw.
💡
Le VNC passe par le tunnel SSH. Si vous fermez votre terminal (celui avec openclaw@srv...), le VNC ne marchera plus. Gardez le terminal ouvert tant que vous utilisez le VNC.
⏳
Rappel : le VNC est OPTIONNEL. Ne bloquez pas dessus. Si ça ne marche pas après 5 minutes d'essais, passez à l'Étape N°2. Tout peut se faire via le terminal SSH + Telegram.

Troubleshooting
Les problèmes les plus fréquents et comment les régler

Installation Ubuntu

▶ Le statut reste bloqué / ne passe pas à "Running"
Attendez 10 minutes. Si rien ne bouge, essayez de redémarrer le VPS depuis le panel Hostinger (bouton Restart).

Terminal Hostinger

▶ Le terminal ne s'ouvre pas / écran blanc
Rafraîchissez la page Hostinger. Si ça persiste, essayez un autre navigateur (Chrome recommandé).
▶ Le terminal affiche "connecting..." indéfiniment
Le VPS n'est peut-être pas encore prêt. Vérifiez que le statut est bien "Running". Sinon, redémarrez le VPS.

Script d'installation

▶ "Password truncated to the length of 8"
C'est normal ! VNC limite les mots de passe à 8 caractères. Ce n'est pas une erreur.
▶ apt-get freeze / "dpkg lock"
Tapez : sudo rm /var/lib/dpkg/lock-frontend && sudo dpkg --configure -a
▶ Le script s'arrête avec une erreur rouge
Relisez le message d'erreur. Souvent c'est un problème de connexion réseau. Relancez le script. Si ça persiste, redémarrez le VPS et recommencez.

Connexion SSH

▶ "connection refused" ou "connection timed out"
Vérifiez l'IP dans hPanel Hostinger et que le VPS est bien "Running". Vérifiez aussi que vous utilisez le bon mot de passe.
▶ "Permission denied"
Mauvais mot de passe. Retapez-le soigneusement (rappel : rien ne s'affiche quand vous tapez). Si ça ne marche toujours pas, réinitialisez le mot de passe depuis le panel Hostinger.

Bureau distant (VNC)

▶ vnc.html affiche une page blanche ou "connection refused"
websockify n'est pas lancé. Dans votre terminal SSH, tapez :
websockify --web /usr/share/novnc 6080 localhost:5901 &
▶ Écran gris ou noir dans noVNC
Faites vncserver -kill :1 puis vérifiez ~/.vnc/xstartup et relancez vncserver :1 -geometry 1280x720
▶ Chrome crash immédiatement dans le VNC
Pas assez de RAM ou problème sandbox. Lancez avec : google-chrome --no-sandbox --disable-gpu
▶ Le VNC ne marche pas du tout
Pas de problème. Passez directement à l'Étape N°2. Tout peut se faire via le terminal SSH + Telegram.
