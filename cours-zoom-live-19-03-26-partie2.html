Créer un site d'artiste avec Claude

Claude (via le mode Cowork) peut créer un site web d'artiste complet en quelques minutes. Il suffit de lui donner les bonnes instructions et il génère un fichier HTML 100% autonome, prêt à être hébergé.

Le processus étape par étape

Étape 1 — Décrire ce que vous voulez

Dites à Claude : « Crée-moi un site web d'artiste pour [Nom]. C'est un artiste [genre musical] basé à [ville]. Il faut une section biographie, une section musique avec liens d'écoute (Spotify, Deezer, Apple Music), une section vidéos, une section dates de concerts, et un formulaire de contact. »

Étape 2 — Donner de l'inspiration visuelle

Envoyez des captures d'écran de sites web que vous aimez. Claude peut s'en inspirer pour le design. Vous pouvez aussi utiliser des templates de sites comme Framer pour trouver des mises en page attractives.

Étape 3 — Claude génère le site

Claude crée un fichier HTML complet avec le design, les textes, les liens, et toute la structure. Le fichier est autonome : pas besoin de serveur spécial, juste un hébergement de fichiers statiques.

Étape 4 — Itérer et corriger

Demandez des modifications à la voix ou par écrit : « Change la couleur de fond », « Ajoute une section FAQ », « Mets la photo de couverture en plein écran ». Claude modifie le site en temps réel.

Étape 5 — Vérifier les liens

Demandez à Claude de scanner le web pour trouver les vrais profils de l'artiste (Spotify, Deezer, Apple Music, YouTube, Instagram) et de mettre les bons URL automatiquement.

💡
Astuce : donner le code source en plus du screenshot

Si vous aimez le design d'un site existant, vous pouvez faire un clic droit → « Afficher le code source » et envoyer ce code à Claude en plus de la capture d'écran. Il aura ainsi à la fois la référence visuelle ET la structure technique pour reproduire le design fidèlement.

3

Héberger gratuitement sur GitHub Pages

GitHub Pages est un service d'hébergement gratuit proposé par GitHub. Il permet de publier des sites web statiques (HTML, CSS, JS) directement depuis un dépôt GitHub. C'est gratuit jusqu'à 1 000 pages et largement suffisant pour un site d'artiste.

Procédure complète : de zéro à en ligne

1

Créer un compte GitHub

Rendez-vous sur github.com et créez un compte gratuit. Choisissez un nom d'utilisateur simple (idéalement le nom de votre projet ou label).

2

Créer un « repository » (dépôt)

Cliquez sur le bouton « New » (en vert). Donnez un nom à votre dépôt (ex : monsite ou nom-artiste). Cochez « Public » (obligatoire pour GitHub Pages gratuit). Cliquez sur « Create repository ».

3

Uploader votre fichier HTML

Dans votre dépôt vide, cliquez sur « uploading an existing file ». Glissez-déposez le fichier HTML que Claude a créé pour vous. Le fichier principal doit s'appeler index.html pour être la page d'accueil. Cliquez sur « Commit changes » (le bouton vert en bas).

4

Activer GitHub Pages

Allez dans l'onglet « Settings » (engrenage en haut). Dans le menu de gauche, cliquez sur « Pages ». Sous « Build and deployment », dans Source, sélectionnez « Deploy from a branch ». Dans Branch, sélectionnez « main » et laissez le dossier sur « / (root) ». Cliquez sur « Save ».

5

Attendre 1 à 2 minutes

GitHub construit votre site. Après 1-2 minutes, un message apparaît en haut : « Your site is live at https://votre-username.github.io/nom-du-depot/ ». C'est votre site en ligne !

⚡
Important : le nom du fichier

Pour que votre page d'accueil fonctionne automatiquement, le fichier principal DOIT s'appeler index.html. Si votre fichier s'appelle autrement (ex : site-artiste.html), il faudra ajouter le nom complet dans l'URL : votre-username.github.io/depot/site-artiste.html.

Résultat : URL de votre site

Élément Exemple
Username GitHub monlabel
Nom du dépôt site-artiste
URL générée https://monlabel.github.io/site-artiste/
Coût Gratuit — 0€
Limite Jusqu'à 1 000 pages par dépôt
4

Connecter son nom de domaine

Si vous possédez un nom de domaine (ex : www.monartiste.com), vous pouvez le connecter à GitHub Pages pour que votre site soit accessible via votre propre adresse au lieu de l'URL GitHub par défaut.

Procédure en 2 parties

Partie A — Configuration sur GitHub

1

Aller dans Settings → Pages

Dans votre dépôt GitHub, allez dans Settings (engrenage) puis Pages dans le menu de gauche.

2

Remplir « Custom domain »

Descendez jusqu'à la section « Custom domain ». Tapez votre nom de domaine sans accent (ex : www.monartiste.com). Cliquez sur Save.

3

Cocher « Enforce HTTPS »

Si la case apparaît, cochez « Enforce HTTPS » pour sécuriser votre site avec un certificat SSL gratuit (le petit cadenas dans la barre d'adresse).

⚠️
Attention aux accents dans les noms de domaine

Les noms de domaine n'acceptent pas les accents dans la configuration DNS. Si votre domaine contient des caractères accentués (è, é, à…), utilisez la version sans accent dans la configuration GitHub et DNS.

Partie B — Configuration chez votre hébergeur de domaine

4

Se connecter au panneau DNS

Connectez-vous chez votre registrar (OVH, Ionos, GoDaddy, Hostinger, Google Domains…). Allez dans la section DNS ou Zone DNS de votre domaine.

5

Ajouter un enregistrement CNAME

Créez un nouvel enregistrement DNS avec ces valeurs exactes :

Champ Valeur à saisir
Type CNAME
Nom / Host www
Cible / Value votre-username.github.io
TTL 3600 (ou laisser par défaut)
6

Sauvegarder et patienter

Sauvegardez l'enregistrement DNS. La propagation peut prendre entre 10 minutes et 24 heures. Après ce délai, votre site sera accessible sur www.monartiste.com.

💡
Astuce

GitHub créera automatiquement un fichier CNAME dans votre dépôt après avoir configuré le domaine personnalisé. Ne supprimez pas ce fichier — il est nécessaire au bon fonctionnement.

🌐 Rediriger aussi le domaine « nu » (sans www)

Si vous voulez que monartiste.com (sans le www) fonctionne aussi, ajoutez des enregistrements de type A pointant vers les IP de GitHub Pages :

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

Ces 4 enregistrements A avec le nom @ permettront à votre domaine sans www de fonctionner aussi.

Utiliser des templates (Framer)

Pour obtenir un résultat visuel encore plus professionnel, vous pouvez utiliser des templates de design comme point de départ. Le site Framer propose une marketplace de templates modernes que vous pouvez utiliser comme inspiration pour Claude.

La méthode template → Claude

1. Parcourir la marketplace Framer

Allez sur framer.com/marketplace et parcourez les templates. Cherchez des designs qui correspondent à l'univers visuel de votre artiste (sombre pour le rap/metal, lumineux pour la pop, etc.).

2. Faire une capture d'écran

Quand vous trouvez un design qui vous plaît — même juste une section (header, FAQ, galerie photos…) — faites une capture d'écran de cette partie.

3. Envoyer à Claude

Envoyez la capture à Claude en disant : « Je veux que tu t'inspires de ce design pour créer mon site. » Claude analysera le layout, les couleurs, la typographie et reproduira un style similaire.

4. Bonus : envoyer le code source

Faites un clic droit sur la page → « Afficher le code source de la page ». Copiez-collez ce code à Claude. Combiné avec la capture d'écran, il pourra reproduire le design de manière très fidèle.

📌
Vous pouvez mixer les inspirations

Rien ne vous empêche de prendre le header d'un template, la section FAQ d'un autre, et le footer d'un troisième. Dites à Claude : « Pour le header, inspire-toi de ça [screenshot 1]. Pour la FAQ, fais comme ça [screenshot 2]. » Il assemblera le tout dans un design cohérent.

6

Pages de capture & funnels

Au-delà du site vitrine, Claude peut créer des pages de capture (landing pages) pour collecter des emails, des inscriptions ou des pré-commandes. C'est la base de tout funnel marketing pour un artiste ou un label.

Qu'est-ce qu'une page de capture ?

Une page de capture est une page web avec un seul objectif : récupérer l'email (ou le numéro de téléphone) d'un visiteur en échange de quelque chose de valeur (un morceau exclusif, un code promo, un accès anticipé, etc.).

Ce que Claude peut créer

Type de page Utilité Exemple de contenu offert
Pré-save Collecter des emails avant une sortie Accès anticipé au single, preview exclusive
Opt-in fan Construire une mailing list de fans Morceau inédit gratuit, remix exclusif
Événement Inscriptions à un live, showcase Accès gratuit ou prioritaire à l'événement
Formulaire multi-étapes Qualifier les contacts (booking, presse) Formulaire en plusieurs étapes pour filtrer les demandes
🔑
À retenir

Un site d'artiste sans page de capture, c'est comme une boutique sans caisse. Le visiteur vient, regarde, et repart. Avec une page de capture, vous récupérez son email et pouvez le recontacter pour chaque sortie, chaque date, chaque actualité.

Structure d'une page de capture efficace

1. Un titre accrocheur

Court, direct, qui décrit immédiatement ce que le visiteur va obtenir. Ex : « Reçois mon prochain single en avant-première, avant tout le monde. »

2. Un visuel fort

Photo de l'artiste, artwork du projet, ou vidéo teaser. Le visuel doit capter l'attention en moins de 2 secondes.

3. Un formulaire simple

Le minimum : un champ email + un bouton. Ne demandez pas le nom, le prénom, la date de naissance — chaque champ supplémentaire fait baisser le taux de conversion.

4. Un appel à l'action clair

Le bouton doit dire exactement ce que la personne obtient : « Recevoir le single » plutôt que « Envoyer » ou « S'inscrire ».

7

Connexion Builderall & outils mail

Une fois votre page de capture créée, il faut la connecter à un outil d'emailing pour stocker les emails collectés et envoyer des séquences automatisées. L'outil mentionné dans le live est Builderall, mais le principe s'applique à tous les autorépondeurs.

Les 3 méthodes de connexion

Méthode Difficulté Description
1. Embed du formulaire ⭐ Facile Créez votre formulaire directement dans Builderall, copiez le code d'intégration (embed/iframe), et demandez à Claude de l'insérer dans votre page HTML. C'est la méthode recommandée.
2. Connexion API ⭐⭐ Intermédiaire Claude peut coder un formulaire qui envoie les données directement à l'API de Builderall via un webhook. Plus technique mais plus élégant visuellement.
3. Zapier / Make ⭐⭐ Intermédiaire Utiliser un outil d'automatisation comme Zapier ou Make pour connecter le formulaire HTML à Builderall sans toucher au code.
💡
Demandez à Claude de vous guider

Vous n'avez pas besoin de savoir comment faire la connexion technique. Dites simplement à Claude : « J'utilise Builderall pour ma base mail. Est-ce que tu peux interconnecter le formulaire de capture avec mon outil Builderall ? Si oui, explique-moi comment faire. » Il vous expliquera étape par étape, adaptée à votre niveau.

Autres outils d'emailing compatibles

📧

Builderall

Suite marketing complète avec autorépondeur, tunnels de vente, et constructeur de pages.

📨

Mailchimp

Gratuit jusqu'à 500 contacts. Idéal pour débuter. Formulaires embed faciles à intégrer.

✉️

Brevo (ex-Sendinblue)

Solution française. Plan gratuit généreux (300 emails/jour). Bon pour les artistes francophones.

📬

ConvertKit

Spécialisé pour les créateurs de contenu. Séquences email visuelles et landing pages intégrées.

🔧 Méthode pas à pas : embed Builderall dans votre site

1. Connectez-vous à Builderall → Section « Mailing Boss » ou « Formulaires ».
2. Créez un formulaire de capture avec les champs souhaités (email minimum).
3. Cliquez sur « Obtenir le code d'intégration » ou « Get embed code ».
4. Copiez le code HTML généré.
5. Envoyez-le à Claude en disant : « Intègre ce formulaire dans ma page de capture à l'endroit du bouton d'inscription. »
6. Claude remplacera le formulaire statique par le formulaire Builderall connecté. Les inscriptions arriveront directement dans votre base mail.

8

Veille technologique & MCP

Le monde de l'IA évolue à une vitesse folle. Pour rester compétitif, il est essentiel de faire de la veille technologique et de surveiller les nouveaux outils et connecteurs disponibles.

Les MCP : la clé de l'automatisation

Les MCP (Model Context Protocol) sont des connecteurs qui permettent aux IA comme Claude de se connecter à des outils externes (Canva, Calendly, Google Agenda, Spotify, etc.). Plus un outil a de MCP disponibles, plus il est intéressant à utiliser avec l'IA.

⚡
Pourquoi c'est important pour vous

Si votre outil préféré (Builderall, LearnBox, Distrokid…) n'a pas encore de MCP, Claude ne peut pas y travailler directement à votre place. C'est dans l'intérêt de ces outils de développer leur MCP s'ils ne veulent pas se faire remplacer par des solutions compatibles IA.

Comment faire sa veille

🔍

ChatGPT en mode veille

Configurez ChatGPT pour scanner le web chaque semaine et vous alerter sur les nouveaux MCP et connecteurs disponibles pour vos outils.

🏪

Marketplace OpenClaw

Parcourez la marketplace de skills et connecteurs communautaires. Si quelqu'un a créé exactement ce dont vous avez besoin, importez-le en un clic.

📰

Reddit & newsletters

Suivez les subreddits tech et IA pour être au courant des dernières évolutions. Les mises à jour sont souvent annoncées là en premier.

La vision : tout déléguer à l'IA

L'objectif à terme est de pouvoir déléguer la quasi-totalité des tâches répétitives à l'IA : création de contenu, publication sur les réseaux, gestion des emails, prospection, mise à jour du site web, analyse des stats. L'artiste ou le producteur se concentre sur ce qu'il fait de mieux : créer de la musique et construire des relations humaines.

🔑
À retenir

L'IA ne remplace pas l'humain — elle supprime les tâches chronophages pour que vous puissiez investir votre temps dans ce qui a le plus d'impact. Un site qui prenait une semaine à créer se fait en 10 minutes. Un funnel qui prenait 3 jours se monte en 1 heure. Les 7 heures de travail pour résumer un live de 4 heures deviennent 2 minutes de prompt.

✅ Ce que l'IA fait très bien

  • Créer des sites web complets en quelques minutes
  • Rédiger des posts, articles, descriptions
  • Extraire et structurer des bases de données
  • Automatiser les tâches répétitives
  • Faire de la veille et de la recherche

❌ Ce que l'IA ne fait pas (encore) bien

  • Reproduire votre ton oral / familier à l'identique
  • Remplacer le contact humain dans les relations
  • Faire du montage vidéo avancé
  • Se connecter aux outils sans MCP disponible
  • Prendre des décisions artistiques à votre place
